# Activation Path

## Primary Product Story

From first visit to first value, the product should guide a prospect through one clear path:

1. **Create organization**
2. **Choose assistant template**
3. **Run demo call**
4. **Review workflow + outcome**
5. **Launch live workflow / campaign**

## Primary CTA

Use one dominant CTA across marketing and in-app entry points:

- **Launch your first voice workflow**

Avoid splitting attention across multiple equal-weight calls to action during first-run.

## First-Run UX Rules

- Never start users from a blank state if a template can seed the experience.
- Show one next action at a time.
- Make the first success visible and explicit.
- Keep advanced operator/admin controls out of the primary onboarding path.

## First-Win Events

At least one of these should happen during onboarding:

- demo call completed
- assistant template provisioned
- workflow board populated
- first campaign draft generated

## Outcome Promise

By the end of onboarding, the user should feel:

- “I know what this product does.”
- “I have something configured.”
- “I saw it work.”
- “I know the next step to go live.”

## Packaging Boundary

Keep these layers distinct in copy and navigation:

- **Marketing / Launch** — promise and proof
- **Client Workspace** — activation and daily use
- **Admin Console** — advanced setup and operator controls
